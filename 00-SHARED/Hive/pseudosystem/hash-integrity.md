# Hash Integrity

Every file that passes through the system — evidence, scripts, outputs, source documents — is SHA256-hashed at the moment of access. The hash is embedded in the COC chain entry for that operation. If the file is later modified, the hash will no longer match, and the modification is detectable. This proves to court: "The file examined on date X was this exact file, with this exact content." Hash verification reports are generated by `baseline_reproducibility.py`.

## Flow

**In:** Any file access during agent work — evidence files, script outputs, source documents
**Out:** SHA256 proof embedded in the COC chain entry; verifiable chain linking finding to source file

## Relationships

- [[coc-hash-chain]] — receives SHA256 proofs as part of every tool-call log entry
- [[forensic-layer]] — parent layer; hash integrity is the mechanism of tamper detection
- [[agent-work]] — all file access during work triggers hash computation
- [[human-evidence]] — evidence files in vault should be hash-verified before promotion

## Natural Metaphor

A wax seal on a letter: you cannot open the letter without breaking the seal, and a broken seal proves the letter was read. The hash is the seal; the file is the letter.
