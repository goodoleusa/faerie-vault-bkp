# The Hive

The Hive is the complete faerie agent system — a self-maintaining intelligence architecture designed to investigate, remember, and improve across sessions. It is not a tool or a workflow; it is a living system with a metabolism. Context flows in, knowledge crystallizes out, and forensic provenance anchors every finding to the exact agent state that produced it.

## Flow

**In:** User intent, evidence, external data, prior session handoffs
**Out:** Crystallized knowledge (HONEY), validated findings (NECTAR), court-grade provenance (forensics), human-reviewable drafts (vault)

## Relationships

- [[session-lifecycle]] — the heartbeat: faerie → work → handoff, repeat
- [[memory-layer]] — the long-term nervous system
- [[state-engine]] — the continuity bridge across sessions
- [[agent-layer]] — the workforce: 23 specialist types
- [[forensic-layer]] — the bedrock: immutable, write-only, court-defensible
- [[vault-layer]] — the human interface: Obsidian-readable evidence layer
- [[equilibrium]] — the regulating force that keeps the system balanced

## Natural Metaphor

A beehive: workers, memory, structure, and a queen process (faerie) that orchestrates without micromanaging. The hive thinks collectively; no single agent holds the full picture.
