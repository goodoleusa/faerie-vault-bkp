# Pipeline Design — Vault Meta-Architecture

> *This document explains why the pipeline is designed the way it is.
> It is the human-facing summary of `docs/SCIENTIFIC-RIGOR.md` in the repo.*

---

## The Vault's Role

The Obsidian vault is **not** the investigation's data store. It is the **human annotation
and review GUI** layered on top of an immutable forensic pipeline.

```
RAW DATA          →  Pipeline scripts  →  scripts/audit_results/*.json
                                          (source of truth, git-tracked)
                                                    ↓
                                      vault_narrative_sync.py
                                      vault_nectar_atomize.py
                                                    ↓
VAULT (this)      →  00-SHARED/Agent-Outbox/  (agent drafts, awaiting review)
                  →  30-Evidence/             (human-promoted, annotated)
                  →  Dashboards/              (live Dataview views)
```

Agents write to `00-SHARED/Agent-Outbox/` only. Humans review and promote to `30-Evidence/`.
The vault cannot corrupt the forensic record — it is a read layer.

---

## Why Progressive Disclosure

Each `/data-ingest` run processes a **staged slice** of raw data, not everything at once.

| Stage | Data % | What the agent does | Forensic value |
|---|---|---|---|
| SEED | 15% | Forms first impressions (FIRST_IMPRESSION blocks written) | Impressions timestamped BEFORE confirming data |
| DEEPEN | 50% | Tests seed hypotheses vs. new evidence | Documents whether impressions held or changed |
| EXTEND | 75% | Convergence check | Bayesian stopping signal |
| FULL | 100% | Final pass if not yet converged | Hard stop at max runs |

**Run planner:** `python3 scripts/run_planner.py --status` — shows current stage and estimates remaining runs.

**Convergence check:** After each EXTEND/FULL stage, run:
```
python3 scripts/run_planner.py --converged [H1_Δ H2_Δ H3_Δ H4_Δ H5_Δ]
```

## Why Pre-Registration

`scripts/audit_results/user_hypotheses.json` + `scripts/audit_results/run_plan.json`
are committed to git **before** any substantive analysis runs.

The git timestamp is the pre-registration timestamp. If the analysis validates H2, the
pre-registration proves H2 was predicted before the confirming data was introduced —
the same proof structure as a clinical trial registration.

See `docs/HYPOTHESIS-PREREGISTRATION.md` for the questionnaire.
See `docs/SCIENTIFIC-RIGOR.md` for the full court-admissibility argument.

## Why Agent Independence Is Enforced

Agents improve between sessions via OTJ learning. **The constraint**: improvements are
process-level only (analytical methods, workflow patterns) — never case-specific findings.

`scripts/lint_agent_cards.py` scans all agent cards for entity names, IPs, domains,
or dataset references. Lint failures block the pipeline.

This ensures: an attorney cannot claim "the AI was trained to reach this conclusion."

## Human Roles — What You Can and Cannot Change

| Your action | Effect | Logged how |
|---|---|---|
| Annotate a vault note | Clarifies agent understanding | `ann_hash` + `ann_ts` fields, signed |
| Add new rawdata and queue a new run | Extends coverage | Git commit + new run_plan.json |
| Write a steering note in Human-Outbox | Adjusts agent focus for next run | `type: steering` note with timestamp |
| Flag an agent error in REVIEW-INBOX | Creates sprint task to investigate | HIGH flag, timestamped |
| Promote a finding to 30-Evidence/ | Confirms agent draft is valid | Move action logged in vault |

**Never:** Tell an agent what conclusion to reach. Change hypotheses after seeing results.
Remove data that contradicts hypotheses.

## Pre-Registration Checklist (new investigation)

- [ ] `user_hypotheses.json` completed and committed
- [ ] `run_plan.json` generated by `run_planner.py` and committed
- [ ] `lint_agent_cards.py` passes (no case data in agents)
- [ ] Seed run complete — FIRST_IMPRESSION blocks written (V1 state)
- [ ] At least one DEEPEN run complete (V2 state, delta documented)
- [ ] Convergence checked after each EXTEND/FULL stage
- [ ] Final: `python3 scripts/baseline_reproducibility.py --report`
