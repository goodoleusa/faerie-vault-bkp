# Blueprint: Session Manifest

Atom-style record of a completed session — what ran, what was produced, metrics snapshot.
Machine-generated at /handoff by session_stop_hook or memory-keeper. Consumed by faerie
at next startup and by the Dashboards/session-briefs/ Dataview panel.

One file per session. Not promoted — it's a reference record for the system and for humans
scanning the Dashboards panel.

## Metadata

| Field | Value |
|-------|-------|
| ID | blueprint_session_manifest |
| Output dir | 00-SHARED/Dashboards/session-briefs/ |
| Filename pattern | `{YYYY-MM-DD}-{session_id}-manifest.md` |
| Agent types | memory-keeper, context-manager (auto-generated at session end) |
| Promotion target | None — reference record, not promoted |
| Human annotation | Not expected — machine-consumed |

## Atom Table (Required Body Element)

The manifest body MUST contain a markdown atom table. Labels are fixed — tools parse by label:

```markdown
| Atom | Value |
|------|-------|
| Session ID | {session_id} |
| Date | YYYY-MM-DD |
| Duration | {N} minutes (approx) |
| Agents spawned | {N} |
| Tasks claimed | {task_ids or "none"} |
| Tasks completed | {N} |
| Findings produced | {N} |
| Queue items added | {N} |
| Memory writes | scratch ({N} MEM blocks) / NECTAR ({N} promoted) / HONEY (unchanged|updated) |
| Stream path | ~/.claude/memory/streams/{session_id}/ |
| COC log | {path or "none"} |
```

## Required Sections

### Atom Table
The machine-readable table. Required. Fixed labels. All rows present.

### Session Summary
2-5 sentences. What was the session's primary output? What changed in investigation state?

### Open at Close
What was in-flight at session end. Threads to pick up at next /faerie.

## Structure Rules

- Generated by memory-keeper / session_stop_hook — not hand-written
- `ts` uses ISO8601 with Z suffix (UTC)
- `status: partial` when session ended before /handoff completed
- `status: interrupted` when auto-compact fired without checkpoint being written
- Stream path recorded even if stream is empty
- Atom table labels are fixed strings — downstream tools parse by exact match
- Session ID format: `{YYYY-MM-DD}-{6-char-hex}` (from session_metrics.py)
- File is never edited after creation

## Validation Rules

- `ts` must be valid ISO8601Z
- Atom Table must be present with all 11 required rows
- All 11 atom labels must match exactly
- `status` must be one of: complete, partial, interrupted
- Session Summary section must be non-empty
- No case data in frontmatter fields or atom table values
