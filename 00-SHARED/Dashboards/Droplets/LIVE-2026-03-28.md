## 00:14 — Droplet (inspired) — Stigmergy: Why Agent 2 Read Agent 1's Report

Agent 2 was dispatched to do independent Claude memory research — launched *because* we assumed agent 1 might be lost (it had been submerged for a long time). No coordinator told agent 2 "go check if agent 1 finished." No message was passed between them. Agent 2 was given the same output path, tried to write, checked if the file existed (standard safety behavior), found agent 1's completed 312-line report already there — and *adapted*. Instead of overwriting, it read the prior work and began synthesizing a merged version.

This is **stigmergy** — coordination through environment modification, not direct communication. It's how ant colonies build structures: no ant talks to another ant. Each ant modifies the environment (deposits pheromone, moves dirt), and the next ant reads the environment and acts accordingly. The filesystem IS the pheromone trail. The vault outbox IS the coordination bus.

This validates something deep about faerie's architecture: **you don't need an agent-to-agent messaging protocol if agents write to known paths.** The file system is the message. The act of creating an artifact IS the coordination signal. Agent 2 didn't need to be told "agent 1 finished" — the existence of the file at the expected path *was* the message.

This is also why the two-stage finding architecture (Agent-Outbox → human promotes to 30-Evidence/) works. Agents coordinate through shared space, not shared state. Each agent sees the world as it is NOW — files that exist, paths that are populated — and adapts. No centralized router needed. The environment routes.

**The design principle:** When you want emergent coordination, don't build message buses. Build shared environments with predictable paths. Let the filesystem be the nervous system. The agents are already doing it — we just need to notice and lean into it.

## 00:32 — Droplet (inspired) — Two Layers, Not One: Private Trail + Public Pheromone

Streaming and stigmergy aren't in tension — they're orthogonal layers that serve different masters.

The **private stream** (`streams/{task_id}.jsonl`) is the agent's diary. It serves forensic COC — proving HOW a conclusion was reached. No other agent reads it. It survives crashes. It's the evidence layer.

The **shared output** (Agent-Outbox, scratch, known paths) is the pheromone trail. It serves coordination — letting the NEXT agent discover what THIS agent did. It enables stigmergy. It's the communication layer.

The failure that breaks everything is an agent that only streams privately and writes its shared output at the very end. If it crashes at 80%, the private trail has the reasoning but the shared path has nothing — the next agent starts from zero. Stigmergy requires progressive output at the shared layer: write drafts at milestones with `status: in-progress`, let the filesystem accumulate the collective state.

This maps cleanly to biological precedent. Ants leave pheromone (shared environment modification) AND have internal state (private neural patterns). The pheromone coordinates the colony; the internal state guides the individual. You need both. An ant that only thinks but never deposits pheromone is invisible to the colony. An ant that only deposits but never thinks is a random walk.

**The architecture:** Stream privately for COC. Write publicly for coordination. Progressive milestones bridge the two — each milestone is both a COC checkpoint AND a stigmergy signal. One write, two purposes.
